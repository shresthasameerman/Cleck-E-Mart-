# OCI8 Checkout Transaction Integration Guide

## Overview

The `checkout_process.php` file contains a complete transaction handler for Oracle OCI8. It:

1. Inserts an order and captures the auto-generated `order_id` using `RETURNING...INTO`
2. Inserts all cart items linked to that order
3. Inserts a payment record
4. Commits on success or rolls back on failure
5. Clears the cart after a successful commit

## Key Features

### RETURNING...INTO Clause

The script uses the `RETURNING order_id INTO :new_order_id` clause to capture the auto-generated order ID generated by your database trigger:

```php
$order_sql = "INSERT INTO \"ORDER\" (customer_id, slot_id, order_status, order_date) 
              VALUES (:cust_id, :slot_id, :status, SYSDATE)
              RETURNING order_id INTO :new_order_id";
```

The `oci_bind_by_name` binding for the output parameter:

```php
$new_order_id = null;
oci_bind_by_name($order_stmt, ':new_order_id', $new_order_id, -1, SQLT_INT);
```

After executing and fetching, `$new_order_id` contains the newly inserted order ID.

### Transaction Safety

All operations are wrapped in a try...catch block:
- On success: `oci_commit($conn)` is called to persist all changes
- On any error: `oci_rollback($conn)` reverts all pending changes
- The cart is only cleared after successful commit

### Batch Item Insertion

Order items are inserted in a loop, reusing the same prepared statement with parameter rebinding:

```php
foreach ($cart_items as $item) {
    oci_bind_by_name($item_stmt, ':order_id', $new_order_id, -1, SQLT_INT);
    oci_bind_by_name($item_stmt, ':product_id', $product_id, -1, SQLT_INT);
    // ... more bindings ...
    oci_execute($item_stmt);
}
```

    }
});

## Database Schema Requirements

The code expects these tables:

### ORDER Table
```sql
CREATE TABLE "ORDER" (
    order_id      NUMBER PRIMARY KEY,
    customer_id   NUMBER NOT NULL,
    slot_id       NUMBER NOT NULL,
    order_status  VARCHAR2(20),
    order_date    DATE DEFAULT SYSDATE,
    FOREIGN KEY (customer_id) REFERENCES CUSTOMER(customer_id),
    FOREIGN KEY (slot_id) REFERENCES SLOT(slot_id)
);

-- Trigger to auto-generate order_id
CREATE SEQUENCE order_seq START WITH 1 INCREMENT BY 1;

CREATE TRIGGER order_id_trigger
BEFORE INSERT ON "ORDER"
FOR EACH ROW
BEGIN
    IF :NEW.order_id IS NULL THEN
        SELECT order_seq.NEXTVAL INTO :NEW.order_id FROM DUAL;
    END IF;
END;
/
```

### ORDER_ITEM Table
```sql
CREATE TABLE ORDER_ITEM (
    order_item_id  NUMBER PRIMARY KEY,
    order_id       NUMBER NOT NULL,
    product_id     NUMBER NOT NULL,
    quantity       NUMBER(10,2),
    unit_price     NUMBER(10,2),
    FOREIGN KEY (order_id) REFERENCES "ORDER"(order_id),
    FOREIGN KEY (product_id) REFERENCES PRODUCT(product_id)
);
```

### PAYMENT Table
```sql
CREATE TABLE PAYMENT (
    payment_id       NUMBER PRIMARY KEY,
    order_id         NUMBER NOT NULL,
    amount_paid      NUMBER(10,2),
    payment_method   VARCHAR2(20),
    payment_status   VARCHAR2(20),
    payment_date     DATE DEFAULT SYSDATE,
    FOREIGN KEY (order_id) REFERENCES "ORDER"(order_id)
);
```

## Error Handling

The code returns a JSON response with this structure:

**Success:**
```json
{
    "success": true,
    "order_id": 12345,
    "message": "Order #12345 created successfully and payment recorded."
}
```

**Failure:**
```json
{
    "success": false,
    "order_id": null,
    "message": "Checkout failed: Failed to insert order: ORA-00001 unique constraint violated"
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `order_id` is NULL | Ensure your trigger is creating a sequence and assigning it to `:NEW.order_id` |
| "Failed to parse" errors | Check SQL syntax, table names, and column names match your schema |
| "Failed to execute" errors | Check that all input parameters (customer_id, slot_id) exist and foreign keys are valid |
| Cart not clearing | Verify that `getCartItems()` and `clearUserCart()` are implemented correctly for your cart storage method |

## Additional Notes

- Use `SQLT_INT`, `SQLT_FLT`, `SQLT_CHR` bind types to ensure proper type casting
- Always validate inputs on the server side (sanitize customer_id, slot_id, etc.)
- Consider adding logging for audit trails
- Add rate limiting to prevent checkout spam
- Implement session checks to ensure the user is authenticated
